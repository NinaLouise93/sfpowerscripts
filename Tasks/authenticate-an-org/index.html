Authenticate an Org

This task is used to authenticate a Salesforce Org (Prod/Sandbox) for either the purposes of deploying or doing a particular action on the org. The task features the following authentication mechanisms

1. JWT 

The following link provides details on what a JSON Web Token and how it is used to authorize and org. 

https://developer.salesforce.com/docs/atlas.en-us.sfdx_dev.meta/sfdx_dev/sfdx_dev_auth_jwt_flow.htm

2. Credentials

This approach utilizes username/password and an optional security token to authenticate to an org. This task utilizes the sfpowerkit auth:login command for enabling this functionality

Using JWT Based Authentication 

The task has the following variables

1. Secure File for using with JWT Authentication.

Follow the steps mentioned here to create the  Private key and test your authentication locally. Once it is done, store the file as a secure file in your library. Click on this link to read more.

2. Username

The username of the user who is authorized to utilize the connected app 

3. Alias

The alias of the org to be referenced elsewhere

4.  Authenticate this org as a DevHub

Turning on this variable, enables the the org to be authenticated as a DevHub, which allows scratch org’s to be created

5. ClientId

The client id used for the JWT based authentication
